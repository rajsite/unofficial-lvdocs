Assert Real Floating-Point Numeric Type VI

Owning Palette: Assert Type VIs and Functions

Requires: Base Development System

Breaks the calling VI unless the input data is a non-complex floating-point numeric type.

This VI does nothing at run time. Use this VI in conjunction with the Type Specialization structure to customize sections of code in a malleable VI (.vim) for non-complex floating-point numeric types or to force a malleable VI to accept non-complex floating-point numeric types only.

 Add to the block diagram  Find on the palette
value in specifies the input data.

This input accepts any non-complex floating-point numeric data type.
value out returns the input data if value in is a non-complex floating-point numeric type. Otherwise, this output returns a double-precision floating-point number 0.
