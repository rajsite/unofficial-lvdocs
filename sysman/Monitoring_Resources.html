Monitoring Real-Time Target Resources (Real-Time Module)

Use the CPU/Memory page of a real-time target probe to monitor the CPU and memory resources of a real-time target.

Note You must install System State Publisher on the real-time target to monitor the target's CPU and memory resources.

Complete the following steps to monitor the CPU and memory resources of a real-time target.

  1. Click a real-time target in the system tree to display a target probe in the Auto View. You also can right-click a target and select Probe to display a target probe in a separate window.
  2. Click the CPU/Memory tab to display CPU and memory statistics.

The System Manager displays a CPU chart for each CPU in the system, labeled by CPU index. Each graph tracks the utilization of the CPU, as a percentage of capacity, over time. For each CPU, the System Manager also displays the percentage of total CPU utilization devoted to each priority level, as well as the percentage devoted to interrupt service routines (ISRs) and Timed Structures. The System Manager also displays a Memory chart that tracks memory usage, in megabytes, over time.
